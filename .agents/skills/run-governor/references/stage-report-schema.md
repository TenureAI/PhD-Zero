# Stage Report Schema

Stage report detail should scale with task complexity, but keep these minimum fields:

1. Stage objective
2. Actions taken (important commands/edits)
3. Evidence observed (logs/metrics/diffs)
4. Risks and likely impact
5. Decisions made
6. Next step
7. Paths to artifacts and logs

## reports/index.md entry

```markdown
| Stage | Status | Report Path | Last Updated |
|---|---|---|---|
| Stage 1 - Intake | done | reports/stage-01-intake.md | 2026-03-01T19:00:00Z |
```
